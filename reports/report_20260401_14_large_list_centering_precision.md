# Android TV 대규모 프로필 리스트 중앙 정렬 정밀 보정 보고서

## 1. 개요
프로필 개수가 많아질 때(5개 이상) 리스트의 뒤쪽 항목들이 화면 중앙에 정확히 정렬되지 않던 문제를 해결하기 위해 레이아웃 제약을 정밀하게 조정하였습니다.

## 2. 주요 개선 사항
### 2.1 상위 컨테이너 정렬 제약 해제 (Alignment Constraint Removal)
- **현상**: 프로필 리스트(`Row`)를 감싸는 부모 `Box`에 `contentAlignment = Alignment.CenterStart`가 설정되어 있었습니다. 이로 인해 리스트가 특정 오프셋 이하(화면 왼쪽 밖)로 이동하려 할 때 부모의 정렬 로직과 충돌하여 오프셋 애니메이션이 비정상적으로 작동하거나 이동이 제한되는 버그가 확인되었습니다.
- **조치**: 부모 `Box`의 명시적인 `contentAlignment`를 제거하였습니다. 이를 통해 `Row`는 계산된 수동 오프셋 값에만 의존하여 화면 좌표계 상에서 자유롭게(음수 좌표 포함) 이동할 수 있게 되었습니다.

### 2.2 대규모 리스트(7~10개) 중앙 정렬 무결성 확보
- 가로 너비 무제한 설정(`unbounded = true`)과 상위 정렬 제약 해제가 결합되어, 프로필 개수가 몇 개이든 상관없이 **포커스된 인덱스에 따른 정확한 중앙 정렬**이 보장됩니다.
- 7명의 캐릭터(지은~시우) 및 그 이상의 테스트 데이터 로드 시에도 완벽한 시각적 균형을 유지합니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 4번(7개) 또는 5번(10개) 키를 눌러 대량의 프로필을 생성한 후, 마지막 프로필까지 이동했을 때 해당 카드가 화면 정중앙에 위치하는지 확인하십시오.
