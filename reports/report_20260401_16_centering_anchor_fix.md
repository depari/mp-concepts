# Android TV 프로필 리스트 좌측 치우침 버그 수정 및 정렬 고도화 보고서

## 1. 개요
프로필 개수가 늘어남에 따라 리스트의 중앙 정렬 기준점이 미세하게 좌측으로 치우치던 현상을 해결하기 위해, 레이아웃 앵커(Anchor) 시스템을 재설계하고 좌표 무결성을 확보하였습니다.

## 2. 주요 수정 사항
### 2.1 레이아웃 기준점(Anchor) 절대 고정
- **현상**: 리스트를 감싸는 부모 `Box`의 정렬 속성이 정의되지 않았을 때, 안드로이드 Compose 엔진이 자식(`Row`)의 너비 변화에 따라 내부 기준점을 유동적으로 조정하면서 수동 오프셋 계산값과의 간섭이 발생하였습니다. 이로 인해 인덱스가 높아질수록 오차가 누적되어 좌측으로 치우치는 현상이 나타났습니다.
- **조치**: 부모 `Box`에 **`contentAlignment = Alignment.TopStart`**를 명시적으로 부여하였습니다. 이를 통해 리스트의 물리적 시작점(`x=0, y=0`)이 시스템적으로 화면 왼쪽 상단 끝에 강력하게 고정되었습니다.

### 2.2 수동 오프셋 계산의 정밀도 정규화
- 기준점이 화면 왼쪽 끝으로 고정됨에 따라, `screenWidth / 2`를 기반으로 한 오프셋 계산 로직이 아무런 외부 간섭 없이 픽셀 단위로 정확하게 적용됩니다.
- 이제 7개, 10개 등 대량의 프로필 환경에서도 모든 카드가 한 치의 오차 없이 화면 정중앙에 수직/수평 균형을 이루며 배치됩니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 4번(7개) 키를 눌러 다량의 프로필을 생성한 후, 마지막 캐릭터까지 이동했을 때 해당 아바타가 화면 중앙 가이드 라인과 정확히 일치하는지 확인하십시오.
