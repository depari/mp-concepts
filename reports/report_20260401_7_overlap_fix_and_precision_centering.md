# Android TV 가이드 겹침 이슈 해결 및 정밀 중앙 정렬 보고서

## 1. 개요
전면 레이어 배치로 인해 발생했던 타이틀/가이드와 프로필 카드의 겹침 현상을 구조적으로 해결하고, 다양한 너비의 아코디언 카드에 최적화된 수동 중앙 정렬 로직을 재정립하였습니다.

## 2. 주요 개선 사항
### 2.1 수직 스택 구조(Column)로 겹침 원천 차단
- **구조 개편**: `Box` 내 중첩 방식 대신 `Column` 기반의 수직 스택 구조를 채택하였습니다.
  - **상단**: `Spacer`와 `Column`을 이용한 타이틀 영역 고정.
  - **중앙**: `weight(1f)`를 부여한 가변 영역에 프로필 카드 배치.
  - **하단**: 하단 여백을 확보한 조작 가이드 배치.
- **결과**: 카드를 전면 레이어로 두지 않아도 물리적으로 영역이 분리되어 겹침 현상이 더 이상 발생하지 않습니다.

### 2.2 가변 너비 대응 정밀 수동 중앙 정렬 (Precision Manual Offset)
- **TvLazyRow 한계 극복**: 웹 프로젝트의 아코디언은 포커스 시 너비가 동적으로 변하기 때문에, 시스템 엔진(`TvLazyRow`)만으로는 수치적 정중앙을 유지하기 어려운 점이 있었습니다.
- **수동 계산식 재도입**: 화면 너비와 아이템의 확장 너비(450dp)를 기반으로 **포커스된 아이템의 중심점이 화면의 중심점과 정확히 일치**하도록 계산하는 로직을 적용하였습니다.
  - `Offset = (screenWidth/2) - (expandedWidth/2) - (이전 아이템들의 너비 합)`
- **동작**: 프로필 이동 시 목록 전체가 수학적으로 계산된 위치로 유연하게 슬라이딩되어 완벽한 중앙 정렬 조작감을 제공합니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 상단 문구와 하단 가이드가 카드와 겹치지 않는지, 그리고 어떤 프로필을 선택해도 화면 중앙에 정확히 위치하는지 확인하십시오.
