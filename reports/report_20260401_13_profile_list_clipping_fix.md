# Android TV 프로필 리스트 잘림(4개 제한) 버그 수정 보고서

## 1. 개요
프로필 개수를 4개 이상으로 설정했을 때 리스트가 화면 너비 제약에 걸려 일부 항목만 표시되고 탐색이 불가능했던 버그를 수정하였습니다.

## 2. 주요 수정 사항
### 2.1 가로 레이아웃 제약 해제 (Unbounded Horizontal Layout)
- **현상**: `Row` 컴포저블이 부모 `Box`의 가로 너비(화면 너비)에 묶여 있어, 4개 이상의 아이템이 존재할 경우 가로 너비를 초과하는 나머지 아이템들이 생성되지 않거나 클리핑되는 현상이 발생하였습니다.
- **조치**: `Row`의 Modifier에 **`wrapContentWidth(unbounded = true)`**를 적용하였습니다. 이를 통해 `Row`는 화면 너비와 상관없이 자식 아이템들의 총합 너비만큼 자유롭게 확장될 수 있습니다.

### 2.3 동적 탐색 및 오프셋 애니메이션 연동
- 레이아웃 제약이 해제됨에 따라, 5개, 7개, 10개 등 대량의 프로필이 로드되어도 모든 아이템이 메모리에 정상적으로 배치됩니다.
- 기존의 수동 오프셋(`animatedOffset`) 로직과 결합되어, 어떠한 인덱스의 프로필을 선택하더라도 화면 중앙으로 정확하게 이동하며 모든 프로필을 탐색할 수 있게 되었습니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 숫자 키(1~5)를 눌러 프로필 개수를 7개 또는 10개로 늘린 후, 끝까지 가로로 이동하며 모든 프로필이 정상적으로 표시되는지 확인하십시오.
