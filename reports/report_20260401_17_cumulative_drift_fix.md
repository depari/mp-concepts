# Android TV 대규모 프로필 정렬 누적 오차(치우침) 최종 해결 보고서

## 1. 개요
프로필 개수가 늘어날 때마다 포커스된 항목이 정중앙에서 좌측으로 미세하게 치우치던 현상을 해결하기 위해, 레이아웃 엔진의 픽셀 보간 로직을 전면 개편하였습니다.

## 2. 주요 수정 사항
### 2.1 픽셀 기반 절대 좌표 연산 (Pixel-Precision Offset)
- **원인 분석**: 기존 Dp 기반 연산은 안드로이드 시스템이 픽셀로 변환하는 과정에서 소수점 버림 현상이 발생합니다. 특히 프로필 리스트와 같이 아이템이 많고 각 아이템마다 '너비 + 간격'이 반복되는 구조에서는 인덱스가 높아질수록 이 미세한 픽셀 오차가 누적되어 눈에 띄는 '좌측 치우침' 현상을 유발하게 됩니다.
- **조치**: 모든 레이아웃 상수(노멀 너비, 확장 너비, 간격)를 `LocalDensity`를 사용하여 **사전에 픽셀(Px) 단위로 변환**한 후 정수 연산을 수행하도록 로직을 교체하였습니다.

### 2.2 부모 레이아웃 앵커 복구 및 배치 교정
- **수직 정렬 안정화**: `Box`의 `contentAlignment`를 `CenterStart`로 복구하여 프로필 리스트의 수직 중앙 정렬을 회복하였습니다.
- **배치 스코프 교정**: `Modifier.layout` 블록 내에서 `Placeable.PlacementScope`를 올바르게 사용하여, 우리가 계산한 정밀 픽셀 오프셋이 레이아웃 시스템의 간섭 없이 절대 좌표로 적용되도록 구현을 완성하였습니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 5번(10개) 키를 눌러 대량의 프로필을 생성한 후, 마지막 10번째 프로필까지 이동했을 때 해당 아바타가 화면 중앙에 완벽하게 안착되는지 확인하십시오. 인덱스 1번부터 10번까지 일관된 정렬 무결성을 보장합니다.
