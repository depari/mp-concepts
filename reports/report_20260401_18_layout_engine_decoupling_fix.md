# Android TV 대규모 리스트 정렬 및 시스템 클리핑 버그 최종 수정 보고서

## 1. 개요
프로필 개수가 5개 이상일 때 정렬이 어긋나고, 7개 이상일 때 리스트 전체가 화면 왼쪽 끝에 고정되어 버리던 현상을 레이아웃 시스템 엔진 레벨에서 해결하였습니다.

## 2. 주요 수정 사항
### 2.1 레이아웃 사이즈 보고 분리 (Layout Size Decoupling)
- **현상**: 프로필이 7개 이상이면 리스트(`Row`)의 총 너비가 화면 너비를 크게 초과합니다. 안드로이드 Compose의 기본 레이아웃 엔진은 자식의 너비가 부모의 제약(Constraints)을 넘어서면 해당 아이템을 화면 경계에 맞추려 하거나, 시작 지점을 강제로 고정하여 "잘림(Clipping)"과 "정렬 오류"를 유발합니다.
- **조치**: `Modifier.layout` 블록 내에서 레이아웃 자신의 사이즈를 **화면 전체 크기(`maxWidth, maxHeight`)**로 고정하여 보고하였습니다.
- **결과**: 시스템은 이 컴포넌트가 화면에 딱 맞는 크기라고 판단하여 간섭을 멈춥니다. 그 상태에서 실제 목록(`placeable`)만 우리가 계산한 음수 오프셋 지점에 절대 좌표로 배치함으로써, 아무리 긴 리스트라도 자유로운 수평 탐색과 정밀한 중앙 정렬이 가능해졌습니다.

### 2.2 픽셀 기반 고정 오프셋 무결성
- 이전 단계에서 적용한 픽셀(Px) 단위 정밀 연산과 결합되어, 3개, 5개, 7개, 10개 등 어떠한 프로필 개수 환경에서도 포커스된 항목이 화면 픽셀 정중앙에 한 치의 오차 없이 배치됩니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 4번(7개) 또는 5번(10개)을 눌러 대량의 리스트를 생성한 후, 마지막 프로필까지 이동했을 때 왼쪽 붙음 현상 없이 부드럽게 정중앙에 정렬되는지 확인하십시오.
