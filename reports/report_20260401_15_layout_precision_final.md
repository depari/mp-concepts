# Android TV 7개 이상 프로필 중앙 정렬 정밀 보정 보고서

## 1. 개요
프로필 개수가 7개 이상으로 늘어날 때 발생하던 중앙 정렬 오차 문제를 해결하기 위해, 시스템 레이아웃 엔진의 자동 배치를 우회하고 직접 좌표를 제어하는 수동 레이아웃(Custom Layout) 로직을 적용하였습니다.

## 2. 주요 수정 사항
### 2.1 수동 레이아웃 측정 및 배치 (Custom Layout Pass)
- **현상**: `Modifier.offset`은 단순 시각적 이동이어서, 프로필 리스트가 길어질 경우 안드로이드 시스템이 가로 너비 제약(Constraints)에 따라 배치를 제한하거나 잘라내는 현상이 발생하였습니다.
- **조치**: `Modifier.layout`을 도입하여 측정(`measure`) 단계에서 가로 너비 제약을 무한대(**`Constraints.Infinity`**)로 확장하였습니다. 이를 통해 7개 이상의 모든 프로필이 메모리 상에서 완벽하게 그려집니다.

### 2.2 픽셀 단위 정밀 오프셋 강제 (PlaceRelative)
- **배치 로직**: `Density` 스코프를 활용하여 `animatedOffset`(Dp)을 실제 기기의 픽셀(Px)로 오차 없이 변환하였습니다.
- **강제 배치**: `placeRelative` 함수를 사용하여 우리가 계산한 수학적 중앙 정렬 위치에 리스트를 절대 좌표로 배치함으로써, 부모 컨테이너의 정렬 로직에 상관없이 포커스된 항목이 화면 정중앙(Pixel-perfect center)에 안착되도록 하였습니다.

### 2.3 안정적인 가시 영역 확보
- 상하단 엣지 문구(25dp/20dp)와 프로필 카드 리스트가 겹치지 않도록 카드 내부 배치를 조정한 이전의 최적화 사항들이 수동 레이아웃 환경에서도 완벽하게 유지됨을 확인하였습니다.

## 3. 결과 확인
- 빌드 결과: **BUILD SUCCESSFUL**
- 확인 사항: 리모컨 4번(7개) 키를 눌러 7명의 캐릭터(지은, 민준, 하나, 서윤, 도윤, 하윤, 시우)를 생성한 후, 마지막 '시우' 캐릭터까지 이동했을 때 해당 아바타가 화면 정중앙에 선명하게 위치하는지 확인하십시오.
