# PIG Player 및 홈 화면 배경 연동 버그 수정 보고서

**날짜**: 2026-03-29
**Task**: pig_player_bug_fix
**담당**: Antigravity

## 1. 개요
기존에 구현된 PIG(Picture-In-Graphic) 모드가 정상적으로 동작하지 않는 문제점을 발견하고, TDD 기반으로 이를 수정하였습니다. 주요 문제는 상태 전이 시 UI 컴포넌트의 렌더링 조건 누락 및 프로필 배경 싱크 불일치였습니다.

## 2. 주요 수정 내용

### 2.1 렌더링 조건 보완 (`App.svelte`)
- **문제**: APP_STATE가 `PIG` 모드일 때 `SamsungTVHome` 컴포넌트가 렌더링 조건에서 누락되어 있어, PIG 전환 시 플레이어가 화면에서 사라지는 현상이 발생했습니다.
- **해결**: `APP_MODES.PIG` 상태에서도 `SamsungTVHome` 라이브러리가 유지되도록 렌더링 조건을 확장하였습니다.

### 2.2 포커스 시스템 고도화 (`homeNavigationStore.ts`, `SamsungTVHome.svelte`)
- **수정**: `homeFocusStore`의 `SECTIONS`에 'pig' 섹션을 추가하여 네비게이션 시스템에 통합하였습니다.
- **자동화**: PIG 모드 진입 시 별도의 조작 없이도 PIG 플레이어가 바로 포커스되어 사용자가 즉시 전체 화면(`Enter`)으로 전환할 수 있도록 자동 포커스 기능을 추가하였습니다.

### 2.3 배경 프로필 동기화 (`ProfileDashboard.svelte`)
- **동기화**: 프로필 대시보드에서 컨텐츠 선택 시, 홈 화면 배경을 담당하는 `selectedIndex`가 즉시 업데이트되도록 `selectProfile()` 호출을 추가하였습니다. 이로써 PIG 재생 시 해당 유저의 개인화된 홈 화면이 배경으로 노출됩니다.

### 2.4 컨텐츠 데이터 전달 강화
- **수정**: `triggerDeepLink` 호출 시 텍스트 ID뿐만 아니라 전체 컨텐츠 객체를 전달하여, PIG 플레이어에서 타이틀, 프로그레스, 썸네일 그라데이션이 정확하게 표시되도록 개선하였습니다.

## 3. 테스트 결과 (TDD)
- **추가된 테스트 케이스**:
  - `APP_STATE`가 PIG일 때 PIG 플레이어 레이어 정상 렌더링 확인 (Pass)
  - 선택된 프로필 변경 시 홈 화면 배경(Accent Color) 동기화 확인 (Pass)
- **전체 유닛 테스트 관통**: 6개 파일, 27개 테스트 모두 성공.

## 4. 최종 결과물
- 사용자가 프로필 선택 화면이나 대시보드에서 컨텐츠를 선택하면:
  1. 즉시 해당 유저의 테마를 가진 홈 화면 배경이 나타납니다.
  2. 2초간의 딥링크 로딩 후, 우측 상단에 PIG 플레이어가 활성화됩니다.
  3. PIG 플레이어는 자동으로 포커스(테두리 강조)되며, 'Enter' 키로 자연스러운 전체 화면 전환이 가능합니다.

이로써 PIG Player 요구사항을 완벽히 충족하도록 기능을 개선하였습니다.
