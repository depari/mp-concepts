# 스마트 TV 프로파일 확장 기능 개발 계획서 (Phase 2)

## 1. 개요
현재 성공적으로 구현된 스마트 TV 프로파일 선택 화면 POC(Phase 1)를 기반으로, 실제 TV의 OS 사용 환경과 콘텐츠 서비스 연동을 고려한 고도화 기능(Phase 2) 개발 계획입니다. 
본 계획서는 **1) 미니 모드(Partial UI)**, **2) 프로필별 전체화면 맞춤 콘텐츠 미리보기/탐색**, **3) 콘텐츠 Preview/Browse 프레임워크 지원** 구조 구축을 주요 목표로 합니다.

---

## 2. 주요 개발 목표 및 세부 요구사항

### 2.1. 프로파일 미니 모드 (Mini Mode / Partial UI)
기존 시청 경험을 단절시키지 않고, 백그라운드 화면 오버레이를 통해 빠르고 가볍게 프로필을 스위칭할 수 있는 퀵 패널 기능을 제공합니다.
* **Partial UI 오버레이**: TV Home 메인 화면이나 라이브러리, 라이브 TV 화면 위에 단축키 호출 즉시 화면의 특정 영역(하단이나 측면 등)에 최적화된 작은 크기의 프로필 바가 팝업됩니다.
* **단축키(Hot-key) 매핑**: 사용자가 복잡한 메뉴 뎁스를 통하지 않도록 지정된 리모컨 키(웹 환경에서는 임의의 키보드 단축키, 예: `M` 키)를 통해 쉽게 활성화/비활성화 및 즉시 전환을 지원합니다.
* **미니 UX 최적화**: 미니 모드 진입 시 전체 형태가 보이지 않더라도 심볼(아바타 컬러) 및 최소화된 타이포그래피를 사용하여 직관적이고 빠른 탐색 모션을 지원합니다.

### 2.2. 프로파일 전체화면 모드 고도화: 콘텐츠 Preview & 탐색
단순히 계정의 Focus를 보여주는데 그치지 않고, 포커스되는 즉시 "나만을 위한 개인화 대시보드"처럼 동작하도록 확장합니다.
* **Focus 기반 동적 확장**: 패널 포커스 시 즉시 개인별 맞춤 데이터(최근 시청한 TV 프로그램, 중단된 VOD, 취향 저격 콘텐츠 등)를 전체화면/패널확장으로 노출합니다.
* **콘텐츠 심층 탐색**: 선택된 프로필 안에서 곧바로 상/하/좌/우(D-pad) 방향키를 활용해 콘텐츠 카탈로그를 브라우징하고 탐색할 수 있는 내부 Grid Layout이 제공됩니다.

### 2.3. 콘텐츠 Preview & Browse 프레임워크 구조 설계 (서비스 허브화)
각기 다른 서드파티/퍼스트파티 앱(CP, Content Provider)으로부터 시청 데이터를 한데 모아 **통합형 특화 서비스 런처**로써 활용 가능한 백엔드 데이터 프레임워크를 마련합니다.
* **통합 DB (Preview DB)**: 
  * 넷플릭스, 아마존 프라임 비디오, 삼성 TV Plus 등 다양한 CP가 개별적으로 가지고 있는 시청 및 찜 이력 메타데이터(타이틀, 에피소드, 진행도, 썸네일, 딥링크 URL)를 중앙 `Preview DB`에 적재/동기화할 수 있는 표준 규격(인터페이스)을 설계합니다.
* **프로필 기반 데이터 라우팅**:
  * 유저들의 프로필 ID(`Profile ID`)와 결합하여 호출 즉시 개별 프로필 구역에 해당 데이터를 실시간 분배합니다. 
* **서비스 런처(Service Launcher)**:
  * 프레임워크 내에서 콘텐츠 목록을 브라우징하다 `Enter(선택)` 입력 시, 해당 CP 앱의 특정 VOD 지점으로 바로 랜딩(Deep Link) 시킬 수 있는 실행기(Launcher) 역할을 부여합니다.

## 3. 프레임워크 아키텍처 및 시스템 레이어 (Structural Design)

POC 프로젝트에 다음 프레임워크 구조를 논리적으로 분리 적용합니다.

1. **데이터 스키마 및 인터페이스 (Preview Data Layer)**
   * `Provider`: 인터페이스 통일 (id, name, logo)
   * `ContentItem`: `content_id`, `provider_id`, `type`, `title`, `thumbnail_url`, `progress`, `action_deep_link`
   * `PreviewDB_Mock`: 위 스키마를 만족하는 대용량 JSON/Local Store Mock 데이터 구축.
2. **서비스 관리자 (Data Manager Layer)**
   * 프론트엔드 Svelte Store 내부에서 `previewContentStore.js`를 신설하여 `Profile ID`에 따라 Preview DB를 쿼리(필터링)해오는 로직 구현.
3. **UI / View Layer**
   * **View A (`MiniMode.svelte`)**: 화면 최상단 `z-index`를 차지하며 투명한 백그라운드를 갖는 Partial 오버레이 뷰.
   * **View B (`ProfileDashboard.svelte`)**: Focus 상태에서 일정 시간(예: 0.8초) 체류 시, 패널 중앙 집중 UI가 부드럽게 지워지고 콘텐츠 브라우징 모드로 페이드 인 전환.

## 4. 단계별 구현 계획 (Phased Roadmap)

### Phase 1: 아키텍처 및 데이터 프레임워크 Mocking
- **내용**: 다중 CP와 다양한 콘텐츠 타입을 지원하는 Mock DB(`src/lib/data/previewDB.js`)와 표준 Interface 작성.
- **결과물**: 프로파일 스토어 내에서 프로필 전환 시 동기화되는 맞춤형 Content 배열 생태계 구축.

### Phase 2: 포커스 콘텐츠 브라우징 및 런처 모드 개발
- **내용**: 포커스가 한 프로필에 N 밀리세컨드 이상 계속될 때 확장 노출되는 `Full-screen Dashboard` 개발. 상/하 탐색형 컴포넌트 추가 및 "가상 앱 딥링크 라우팅" 시뮬레이션 구현.
- **결과물**: 프로필 간의 탐색 외에도 프로필 **내부(Depth)** 로 들어가 콘텐츠를 탐색하는 키보드(D-pad) 관리 체계 확보.

### Phase 3: 미니 모드 (Partial UI) 플로팅 컴포넌트 개발
- **내용**: Home 화면 위젯 느낌의 팝업형 컴포넌트 구축. `M`키 / 리모컨 특정 단축키 입력을 Listener 처리. 
- **결과물**: 끊기지 않고 즉각적으로 호출·폐기되는 새로운 프로파일 스위칭 UX 확보.

### Phase 4: 폴리싱 및 전환 효과 최적화
- **내용**: '전체 화면 대시보드'와 '일반 프로필 탐색 상태', 그리고 '미니 모드' 이 3가지 View 간의 Context Switching을 매우 부드럽고 가볍게(Vanila CSS 변수 & Svelte Motion) 만들 수 있도록 모션/애니메이션 정교화.

---

## 5. 기술 스택 (유지 및 확장)
- **Framework**: Svelte 4 / Vite 5
- **상태 관리**: Svelte Stores (기존 Store의 모듈식 분리)
- **Data Mocking**: Local JSON Data + JS Promise 기반 Fake API 호출 구조 (실제 DB Fetch 환경 시뮬레이션용)
- **Styling**: 기존 CSS Variables 시스템 유지, 글래스모피즘(Glassmorphism) 기반 미니 모드 컴포넌트 추가 개발

---

## 6. 결론
이번 2차 개발을 통해 Profile 화면은 단순한 사용자 진입로(Entry Gate)를 넘어 스마트 TV의 핵심 **개인화 미디어 허브이자 서비스 런처**로 진화합니다. CP들의 데이터를 효과적으로 취합해 사용자가 가장 짧은 동선으로 자신이 원하던 콘텐츠 소비를 재개할 수 있도록 혁신적인 스마트 TV Profile UX를 설계합니다.
