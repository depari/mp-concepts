# 스마트 TV 프로파일 미디어 허브 개발 계획서 (Phase 3)

## 1. 개요
Phase 2에서 구축된 미니 모드 및 수동 대시보드 트리거 기능을 기반으로, 실제 TV 시청 경험으로 연결되는 **홈 화면 진입 시퀀스**와 **대시보드 내 컨텐츠 브라우징 및 딥링크** 기능을 구현합니다.

---

## 2. 주요 개발 목표
1.  **Samsung TV Home 진입 시퀀스 구현**
    *   선택된 프로필의 개인화된 설정(배경, 추천 앱 등)이 반영된 홈 화면(Mock) 렌더링.
    *   프로필 카드에서 홈 화면으로 빨려 들어가는 듯한 몰입형 전환 애니메이션 적용.
2.  **대시보드 내 컨텐츠 탐색 및 딥링크**
    *   대시보드(상세 항목) 진입 시 컨텐츠 카드 간 포커스 이동 기능.
    *   컨텐츠 선택 시 해당 프로필로 자동 전환 및 서비스 앱으로의 딥링크 실행 시뮬레이션.

---

## 3. 상세 설계 및 구현 계획

### 3.1 어플리케이션 상태 관리 확장 (`appStateStore.js`)
*   `currentMode`: `SELECTION` (프로필 선택), `HOME` (TV 홈 화면)
*   `TV 홈 화면` 상태에서는 상단 GNB와 하단 앱 런처 바가 포함된 Mock UI 노출.

### 3.2 홈 화면 전환 애니메이션
*   **Trigger**: 프로필 패널에서 `Enter` 입력 혹은 대시보드에서 컨텐츠 선택 시.
*   **Animation**:
    *   선택된 프로필 아바타/패널이 화면 전체로 줌(Zoom-in) 되며 페이드 아웃.
    *   TV 홈 화면의 UI 요소들이 `Cubic-bezier` 가속도를 받아 부드럽게 페이드 인.
    *   Svelte의 `crossfade` 혹은 `transition:scale` 등을 활용해 매끄러운 연결성 확보.

### 3.3 대시보드 내비게이션 및 딥링크
*   **Focus 관리**: `ProfileDashboard.svelte` 내부에서 `focusedContentIndex` 상태 관리.
*   **Key Handling**:
    *   대시보드 활성화 상태에서 `ArrowLeft/Right/Up/Down`을 통해 카드 간 포커스 이동.
    *   `Enter`:
        1.  해당 컨텐츠가 속한 프로필로 `activeProfile` 변경.
        2.  `previewDB`의 `action_deep_link` 정보를 기반으로 앱 실행 시뮬레이션(팝업 혹은 홈 화면의 특정 섹션 강조).
        3.  대시보드 닫기 및 홈 화면 진입 수행.

---

## 4. UI/UX 디자인 가이드
*   **TV Home 구성**: 
    - 전용 배경 레이어 (선택된 프로필 컬러 톤 반영).
    - 중앙에 대형 추천 컨텐츠 배너.
    - 하단에 가로로 배열된 서비스 앱 아이콘 리스트 (Netflix, YouTube, TV Plus 등).
*   **인터랙션 피드백**:
    - 대시보드 내 카드 포커스 시 보더 하이라이트 및 스케일 업(1.05x).
    - 홈 진입 시 시각적 깊이감(Depth)을 강조한 Parallax 효과 적용.

## 5. 예상 일정 및 마일스톤

3.  **[Day 3]**: 대시보드 내부 컨텐츠 탐색 로직 및 딥링크 연동 완료.
4.  **[Day 4]**: 전체 시나리오 통합 테스트 및 최종 폴리싱.

## 6. 품질 및 테스트 전략

*   **Deep-link Verify**: 컨텐츠 선택 시 정확한 프로필 전환 및 링크 데이터 전달 확인.
