# 프로파일 패널 레이아웃 고정 및 데이터 로딩 잔상 픽스

* **Date**: 2026-03-28
* **Task Number**: 04
* **Memorized Item**: profile_panel_ux_fixes

## 개요
프로필 초기 포커스 시 프리뷰 데이터 로딩 전/후에 따라 하단 요소들의 높이가 팽창하면서 상단의 닉네임(Name)이 위로 밀려 올라가는 레이아웃 점프 현상이 있었습니다. 
더불어 프로필을 좌/우로 이동 시 300ms 로딩 딜레이 시간 동안 이전 프로파일의 카드 내용이 남아있다가 나중에야 깜빡이며 갱신되는 잔상 현상도 발견되었습니다.

## 문제점 개선 (TDD 사이클 준수)
1. **이름/텍스트 점핑 해결**: 
   - 원인: `ProfilePanel.svelte`의 `.panel__bottom` 클래스가 `bottom: 0`과 가변 높이(Flexible Height)를 가져 내부 `panel__contents` 렌더링 시 여백을 밀어냄.
   - 해결: `.panel__bottom`에 고정 높이 `height: 360px`를 주입하고, `justify-content: flex-start` 속성으로 정렬 기준을 상단으로 고정하여 컴포넌트 유무 및 추가에 상관없이 닉네임 위치가 절대적으로 고정되도록 디자인 안정성을 확보.
2. **이전 데이터 잔상(캐시) 현상 조치**:
   - 원인: `previewContentStore.js`의 `fetchPreviewContent` 함수 실행 시, 비동기 API 모의 딜레이 동안 `loading: true`만 설정되고 `data` 상태는 여전히 이전 상태의 쓰레기 값을 보유.
   - 해결: 함수 호출 진입 시 스토어 업데이트 구문에서 `data: []`로 초기 배열 교체 로직을 삽입. 로딩 중에는 카드가 보이지 않다가 페치 완료 시 새 카드가 뜨도록 깔끔하게 정리.
3. **The Crown 에셋 깨짐 수정**:
   - `previewDB.js`에 설정된 'The Crown' 콘텐츠의 Unsplash 원본 링크가 삭제/변경되어 이미지가 엑박으로 노출되는 것을 확인 후, 다른 정상 고화질 풍경 컷으로 대체 복구함.

## 상세 결과
- 프로필 간 횡이동 시 닉네임 글자는 상시 동일한 세로축(Y축)을 꾸준히 유지합니다. (눈에 띄는 튀는 느낌 방지 완료)
- 방향키 변경 즉시 다른 이의 프로파일이 살짝 보이던 크리티컬 버그(Data Leakage 현상)가 완벽히 소고되었습니다.
- 모든 카드의 썸네일 이미지가 에러 없이 정상 로드됩니다.
